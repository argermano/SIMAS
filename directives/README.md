# Diretivas (`directives/`)

Este diretório guarda as **SOPs em Markdown** que definem:
- Objetivo de cada fluxo
- Entradas necessárias
- Scripts em `execution/` que devem ser usados
- Saídas esperadas
- Principais edge cases

Fluxo recomendado:
1. Criar um arquivo como `nome_fluxo.md` descrevendo o processo.
2. Garantir que existam scripts correspondentes em `execution/`.
3. Sempre atualizar a diretiva quando houver aprendizado novo (limites de API, tempos, erros comuns, etc.).
